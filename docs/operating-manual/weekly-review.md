# Weekly Review

## Goal

Convert seven days of output into a tighter playbook for the next week.

## Inputs

- `data/published/published_posts.jsonl`
- `data/reviews/reviews.jsonl`
- `data/analytics/performance_log.jsonl`
- any manually collected notes from live posting

## Weekly Questions

1. Which topics generated the best combination of reach and qualified response?
2. Which hooks overpromised or underperformed?
3. Which reviewer comments kept repeating?
4. Which posting windows produced the strongest engagement velocity?
5. What should be stopped, doubled down on, or tested next?

## Output

Create a markdown brief using `templates/weekly-learning.template.md` and save the polished version in `examples/` or your working notes system.

The brief must contain:

- top wins
- biggest misses
- new hypotheses
- exact experiments for next week

## Update Targets

If the same issue appears three times, update one of these:

- rubric
- persona instruction
- template
- pipeline stage

Do not rely on memory alone.
