# Daily Run

## Goal

Move one strong idea from inbox to publish-ready package without skipping review.

## Daily Checklist

1. Collect fresh inputs into `data/inbox/` or `data/ideas/ideas.jsonl`.
2. Run idea scoring and shortlist the top 1-3 candidates.
3. Pick one candidate based on score plus strategic timing.
4. Generate 3 hook directions.
5. Build one post draft and one optional thread outline.
6. Route the draft to the right reviewers.
7. Apply edits and run the publish check.
8. Log the outcome in `data/drafts/`, `data/reviews/`, or `data/published/`.

## Daily Output Standard

By the end of a successful daily run, the repo should contain:

- one scored idea
- one draft package
- one review decision
- one explicit publish recommendation

## If Time Is Tight

Prioritize in this order:

1. scoring
2. hooks
3. one strong draft
4. voice review

Do not skip factual review on technical claims just to hit a posting cadence.
