# ADR 003: No Autoposting Yet

## Context

Autoposting is attractive, but it compounds mistakes:

- weak hooks get published faster
- wrong claims go live before review
- strategic drift becomes harder to notice

## Decision

Do not add direct publishing automation yet.

The repository may package final posts, but the final handoff remains manual until:

1. scoring quality is stable
2. review routing is trusted
3. weekly learnings repeatedly improve output quality

## Consequences

- manual publish overhead remains
- operational discipline improves
- future automation can be added on a stronger base
